Agora precisamos ter certeza que nosso VSCode está utilizando o Git Bash como terminal principal. Com o VSCode aberto, siga os passos:

  • O VSCode abrirá uma tela de terminal na parte inferior. Por padrão, se tiver utilizando o Windows, ele abrirá o Power Shell, que é um terminal nativo do Windows. Entendendo os comandos executados

  • O comando git add é usado para adicionar as alterações feitas nos arquivos do repositório Git à área de preparação (também conhecida como "staging area" ou "stage) para que possam ser incluídas no próximo commit. O comando git add pode ser usado para adicionar novos arquivos, atualizar arquivos existentes ou remover arquivos do repositório.
  • O comando git commit é usado para confirmar as alterações feitas nos arquivos do repositório Git que foram adicionados à área de preparação com o comando git add. O comando git commit cria um novo versionamento das alterações e adiciona uma mensagem de commit que descreve as alterações feitas. Cada commit no Git tem um ID exclusivo e uma mensagem que o descreve.
  • O comando git status é um comando utilizado no Git para exibir o estado atual do repositório. Ele mostra quais arquivos foram modificados, quais arquivos estão na área de preparação (ou "staging area") prontos para serem confirmados (com o comando "git commit"), e quais arquivos não foram rastreados pelo Git.

usuário git

Antes de entender os comandos que executamos, vamos fazer duas configurações. Como é a primeira vez que estamos utilizando o git em nossa máquina, precisamos nos identificar para a ferramenta. Utilize o código abaixo, trocando o nome "name" pelo seu nome, e o email "email@email.com" pelo seu email cadastrado na sua conta github.

git config --global user.name "name"

git config --global user.email "email@email.com"
